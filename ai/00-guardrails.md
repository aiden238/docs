# 00-guardrails — Project Guardrails

## 목적/범위
- 본 프로젝트에서 AI 도구(GitHub Copilot, gpt_CODEX, Claude)가 따라야 할 **가드레일**, **출력 형식**, **우선순위**를 정의한다.
- 목표: 일관성 · 재현성 · 온보딩 · 드리프트 방지.

## 우선순위(충돌 시 적용)
1) 사용자의 최신 명시 지시  
2) 본 문서(00-guardrails)  
3) 모델별 시스템 프롬프트(`/docs/ai/10-prompts/*.md`)  
4) C.P.T 핸드오프(`/docs/ai/20-contracts/cpt-handoff.md`)  
5) 결정 로그(`/docs/ai/30-decision-log.md`)  
6) 외부 검색/RAG

## 출력 형식(엄수)
- 코드/설정 패치는 **두 단 형식**만 사용  
  1) 교체할 위치 전체 코드  
  2) 교체할 전체 코드
- 장황한 서론 금지, 요청 범위 이탈 금지.
- 불확실성은 “알 수 없음/근거 부족/추측”으로 명시.
- 최신성 이슈(가격/법/스펙/뉴스)는 확인 후 핵심만 요약(필요시 인용 최소화).
- 비밀정보(키/토큰/개인정보) 요청·노출 금지. 예시 값만 사용.

## 승인 게이트(사전 질문 필요)
- 스키마/마이그레이션, 퍼블릭 API 시그니처, 신규 외부 API, 보안/로그 정책 변경.

## 문맥 유지
- 세션 종료 시 변경·결정·미해결을 **5~10줄**로 `/docs/ai/30-decision-log.md`에 누적.
- 반복되는 규칙은 릴리스마다 본 문서로 “승격”, 로그 중복 정리.
